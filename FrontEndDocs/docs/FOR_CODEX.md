# AIへの指示（ルール）

## 作業プロセス
- 進捗は `docs/PROGRESS.md` にステップごとに記載する。
- 関係ドキュメントは改修後、変更があれば整合性を保つためリアルタイムで更新する。
- 不明確な指示・要求は必ず質問して仕様を確定してから作業する。
- ドキュメント改修や実行の前に、このファイルの指示を必ず再確認する。
- 手順を守れなかった場合は理由と再発防止策を記載する。

## 追加ルール（ミス防止）
- 変更前に影響範囲を一言で確認する（変更ファイル/画面/機能）。
- 実装後に動作確認手順を必ず提示する（実機/ログ/テスト）。
- 仕様変更が入った場合は差分を箇条書きで明示する。
- 既存挙動を変える場合は元に戻す方法も記載する。
- リリース影響（versionCode/Name、applicationIdSuffix）の変更有無を明示する。
- ビルド前の versionCode/versionName カウントアップは、**コード更新がある場合のみ必須**。
- コード更新なしで複数端末に再インストールする場合は、バージョン番号は据え置きでよい。
