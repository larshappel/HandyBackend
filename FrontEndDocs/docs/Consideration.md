# Consideration

## 順不同スキャンのメリット / Pros
- どの順序でも処理できるため作業が止まらない
- 誤フォーカス時の入力ミスを減らせる
- 高速スキャン時のオペレーター負担が軽い
- 端末の操作回数が減り、スループットが上がる
- スキャン順のばらつきに強い

## 順不同スキャンのデメリット / Cons
- 判定ルールに依存するため誤判定リスクがある
- 判定不能時のエラーが増える可能性
- ルールを満たさない現場バーコードに弱い
- エラー原因の切り分けが難しくなる
- ログと説明資料の整備が必要

## 旧仕様（フォーカス欄のみ受け付け、違う種類ならエラー）のメリット / Pros
- 判定ロジックが簡単で明確
- 誤判定が原理的に起きない
- 想定外のバーコードでも誤登録しにくい
- ログ解析がシンプル

## 旧仕様のデメリット / Cons
- フォーカス操作が必要で手間が増える
- 高速スキャン時に入力欠落が起きやすい
- 誤フォーカス時に作業が止まる
- 順不同の現場運用に不向き
